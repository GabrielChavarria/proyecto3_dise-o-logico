## Función
Controla el flujo de ingreso de datos de la calculadora mediante una máquina de estados finitos de 4 estados. Gestiona el ingreso del dividendo (máx 127, 7 bits) y el divisor (máx 31, 5 bits), genera el pulso `valid` para arrancar el pipeline de división y espera la señal `done`.

## Puertos

| Puerto | Dirección | Bits | Descripción |
|---|---|---|---|
| `clk` | input | 1 | Reloj 27 MHz |
| `rst_n` | input | 1 | Reset activo en bajo |
| `tecla_valida` | input | 1 | Pulso de nueva tecla detectada |
| `digito[3:0]` | input | 4 | Dígito decimal ingresado |
| `es_numero` | input | 1 | Indica si la tecla es numérica |
| `confirmar_a` | input | 1 | Tecla A: confirma dividendo |
| `ejecutar` | input | 1 | Tecla B: inicia la división |
| `limpiar` | input | 1 | Tecla D: reinicia |
| `done` | input | 1 | Del divisor: resultado listo |
| `dividendo[6:0]` | output | 7 | Dividendo almacenado |
| `divisor_b[4:0]` | output | 5 | Divisor almacenado |
| `valid` | output | 1 | Pulso de 1 ciclo para arrancar el divisor |
| `estado_dbg[1:0]` | output | 2 | Estado actual para debug/display |

## Diagrama de estados

```
         ─────────────────────────────────────────────
         │                  limpiar (D)               │
         ▼                                            │
    [INGRESO_A] ─── confirmar_a (A) ──► [INGRESO_B] ─┤
         ▲                                   │        │
         │                           ejecutar (B)     │
         │                                   │        │
         │                                   ▼        │
         │                            [CALCULANDO] ───┤
         │                                   │        │
         │                             done (pipeline)│
         │                                   │        │
         │                                   ▼        │
         └──────── limpiar (D) ────── [RESULTADO] ────┘
```

## Comportamiento por estado

| Estado | Valor estado_dbg | Display muestra | Acción al presionar número |
|---|---|---|---|
| INGRESO_A | 0 | dividendo | Agrega dígito al dividendo (máx 127) |
| INGRESO_B | 1 | divisor | Agrega dígito al divisor (máx 31) |
| CALCULANDO | 2 | dividendo | No acepta entradas (espera `done`) |
| RESULTADO | 3 | cociente o residuo | No acepta más dígitos |

## Acumulación de dígitos
```
registro = registro * 10 + digito_nuevo
Ejemplo: 1 → 1, luego 2 → 12, luego 7 → 127
```
