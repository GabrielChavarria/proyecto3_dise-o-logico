# Problemas Encontrados y Soluciones

## 1. División por cero

**Problema:** Si el usuario ingresa 0 como divisor y presiona B, el pipeline ejecuta la división igualmente, produciendo un resultado indefinido (cociente = 0xFF…, residuo = 0).

**Solución propuesta:** Agregar validación en la FSM para no emitir `valid` si `divisor_b == 0`, y mostrar un mensaje de error en el display (p. ej., `Err`).

**Estado:** Pendiente.

## 2. Overflow en acumulación de dígitos

**Problema:** Durante el ingreso del dividendo, si el usuario escribe un número mayor a 127 (o mayor a 31 para el divisor), los dígitos extra deben ser ignorados. El uso de registros temporales más anchos (`reg_a[7:0]`, `reg_b[5:0]`) evita overflow en la comparación antes de truncar.

**Solución implementada:** Se compara `nuevo_a <= 127` y `nuevo_b <= 31` antes de actualizar el registro. Si el nuevo valor excede el límite, el dígito se descarta silenciosamente.
